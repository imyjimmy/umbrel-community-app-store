MGit: (Medical) Git

A Git wrapper for secure, decentralized medical data exchange

What is MGit?

MGit is a specialized Git wrapper built on top of git that extends Git's capabilities with Nostr public key integration. MGit is designed specifically for secure, decentralized medical record management and transmission, giving patients and healthcare providers a verifiable way to store, share, and track medical data.

What is MGit Server?

MGit Server is a self-hosted service that stores a user's Medical Data. It authenticates users via Nostr and serves the MGit Repo back to the authenticated client. MGit Server = MGit + Nostr

How does it all work together?

The MGit repo is--you guessed--a git repo, but its contents are a user's medical data. Together with a client App that understands how to render such data, the user essentially has a digital Medical Binder on their smartphone.

Why Decentralized Medical Records?

Traditional Electronic Medical Record (EMR) systems face significant challenges:

Data Silos

Medical records are fragmented across different providers and systems, making it difficult for patients to access their complete health history.

Lack of Interoperability

Different EMR systems often cannot communicate effectively, leading to repeated tests, treatment delays, and potential medical errors.

Security Concerns

Centralized databases create single points of failure and attractive targets for data breaches.

How MGit Works

MGit leverages Git's distributed version control capabilities and enhances them with cryptographic verification through Nostr public keys:

Nostr Authorship

Each commit is linked to a patient's Nostr public key, establishing data ownership.

Version History

Complete audit trail of all changes to medical records, ensuring accountability and enabling historical views of patient data.

Secure Sharing

Granular permission controls allow patients to share specific portions of their medical history with different providers.

Getting Started

MGit provides a familiar command-line interface for those who already know Git: 

$ mgit config --global user.name "Dr. Jane Smith"
$ mgit config --global user.pubkey "npub19jlhl9twyjajarvrjeeh75a5ylzngv4tj8y9wgffsgu..."

After setup, MGit operates with the same basic commands as Git, but with enhanced security and verification for medical data.

Proof of Concept

This server is running a proof of concept implementation of the MGit system. It demonstrates:

  • Nostr-based authentication and authorization
  • Cryptographically verified medical record commits
  • Secure repository access controls
  • Transparent audit trail of all record changes
